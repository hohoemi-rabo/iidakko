# 開発チケットと Todo 運用

> このルールは `docs/tickets/` 配下のチケットを編集するときだけ読み込まれる。

- 機能・要件ごとの開発チケットは **`docs/tickets/`** に連番付きで置く（`00-overview.md` が一覧・進め方の index）。実装の着手前に該当チケットと `REQUIREMENTS.md`・`docs/ui-references/` の参考画像を確認する。
- **各チケット内の Todo は Markdown チェックボックスで管理する。**
  - 未完了：`- [ ]`
  - 完了したら **`- [×]`** に書き換える（`- [x]` ではなく全角風の `×` を使う）。
  - 例：`- [ ] タブを5つ定義する` → 完了後 → `- [×] タブを5つ定義する`
- 作業を終えたら、対応する Todo を必ず `- [×]` に更新してからコミットする。チケットの進捗はこのチェック状態が唯一の情報源。
