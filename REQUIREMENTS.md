# REQUIREMENTS.md ―「いいだっこ」

飯田市子育て応援サイト（ https://www.city.iida.lg.jp/site/kosodate/ ）を
**スマホで見やすいアプリにしつつ**、**登録不要の「子育て施設マップ」** を目玉機能として加えた、地元特化アプリ。

本書は **商工会議所への持ち込み用ベータ（見た目重視デモ）** の仕様であり、
同時に「自分はこういう地域アプリを作れる」という **技術力のアピール資料** を兼ねる。

---

## 実装状況（このベータの現状）

- スキャフォールド作成済み：Expo + expo-router の5タブ（ホーム / さがす / マップ / イベント / きんきゅう）
- データ投入済み：施設47件（保育園・認定こども園45＋つどいの広場・こども食堂2）、記事34件、イベント・緊急情報
- 仕上げ済み：パステル（コーラル/ピンク）テーマ、カテゴリ別の色付きピン＋凡例、ピンタップで施設カード、りんごアイコン／スプラッシュ
- 残作業（手元で実行）：
  1. `npm install` → `npx expo install --fix`
  2. Google Maps APIキーを app.json に設定（Maps SDK / Geocoding API を有効化）
  3. `GOOGLE_MAPS_API_KEY=... node scripts/geocode.mjs` で施設座標を正確化
  4. `npx expo run:android`（または EAS 開発ビルド）で実機確認
- ビルド手順の詳細は README.md を参照

## 0. このベータの位置づけ（最重要・最初に読むこと）

- ゴールは **「動いて見える・綺麗・分かりやすい」デモ** を最短で作ること。実運用システムではない。
- データは **すべて静的な TypeScript / JSON**。**バックエンド・DB・認証・母子手帳機能は作らない**。
- 採用（役所・商工会議所での合意）が決まってから、Phase 2 でデータ更新の自動化を設計する。
- **過剰実装をしない。** 下記「やらないこと」を厳守し、迷ったら作らない方を選ぶ。
- **サイトを全ページ完全再現しない。** ナビ構造＋各カテゴリの代表数件で「見やすくなった」を示せば十分。

### コンセプト（一文で）
> 飯田の子育て情報を、スマホで見やすく。登録いらずで、預け先・遊び場・相談先が地図でわかる。

### 2つの価値（競合相手が違うので同居できる）
1. **サイトを見やすくする** … 競合は「市のWebサイト」。年齢・目的のタブ整理＋PDFのカード化。
2. **登録不要の子育て施設マップ** … 競合は無し。母子モ（記録アプリ）に無い「場所を探す」体験。★目玉

### やること（Phase 1 = このデモの範囲）
- 5タブ：**ホーム / さがす / マップ / イベント / きんきゅう**
- 「さがす」：年齢⇄目的のセグメント切替 → 情報カード（PDFはネイティブカード化＋公式リンク）
- 「マップ」：施設をカテゴリ色ピンで地図表示（react-native-maps）。地図⇄一覧トグル、カテゴリ絞り込み
- 各施設カードから **「地図で見る（経路）」「電話」「公式ページ」** へワンタップ
- 「イベント」：タグ付きカード（元PDF/SNS情報の集約）
- 「きんきゅう」：♯8000・休日夜間当番医への導線
- 手帳PDF（みんなで子育てナビ / パパナビ / まご手帳）への導線
- パステル（コーラル/ピンク）統一テーマ、大きめ文字・広いタップ領域

### やらないこと（Phase 1 では絶対に作らない）
- ❌ ログイン / 会員登録 / Supabase などのバックエンド
- ❌ 母子手帳・成長記録・予防接種スケジュール（＝母子モの領域。被せない）
- ❌ サイトのスクレイピング・RSS自動取得・データ同期処理
- ❌ プッシュ通知の実配信（UIの見た目のみOK、実装はしない）
- ❌ PDFのアプリ内パース・全文表示（PDFは外部リンクで開くだけ）
- ❌ 実行時ジオコーディング（住所→緯度経度はビルド前に確定し静的に持つ）
- ❌ オンライン予約・申請（公式フォームへのリンクのみ）
- ❌ サイトの全ページ・全記事の網羅（代表コンテンツのみ）

## 2. アピールポイント（このデモが証明すること）

1. **地図アプリが作れる**（react-native-maps、ピン・絞り込み・経路リンク）
2. **散在する地域情報を構造化できる**（サイト/PDF/SNSを1つのデータモデルに整理）
3. **既存サイトのUXを実際に改善できる**（Before=PDF／After=カード を並べて見せる）
4. **登録不要で即UXに到達する設計**（高齢の祖父母でも開いてすぐ使える）
5. **公式情報との正しい連携設計**（出典明記・公式へ誘導・改変しない）
6. **横展開できるテンプレ性**（他自治体へ同型展開可能 ＝ 事業化の芽）

## 3. 技術スタック

- **Expo (React Native) + TypeScript**
- **expo-router**（ファイルベースのタブ・スタックナビ）
- **NativeWind v4**（Tailwind 記法。まさゆきさんの Tailwind 資産を活かす）
- **react-native-maps**（地図表示。Expo の config plugin で導入、**EAS 開発ビルドが必要**。Expo Go では機能制限あり）
  - Android：Google Maps を使用 → **Google Maps APIキー**を `app.json` に設定
  - iOS：Apple Maps（キー不要）でも可
- 外部リンク：`expo-linking`（電話 `tel:`、経路 `https://www.google.com/maps/...`、公式URL、PDF）
- データ：`/data/*.ts`（静的）。型は `/types.ts` に集約
- 配布：EAS Build で Android APK（過去の「パスもん」と同じ要領）

### 地図データの作り方（デモを軽く保つ要）
- 住所 → 緯度経度の変換は **開発時に一度だけ** 行い、結果を `facilities.ts` に**数値で直書き**。
- 実行時に地図APIへ問い合わせない＝**無料・高速・一覧はオフラインでも出る**。
- 保育園一覧の住所は飯田市公式サイトから取得済み。これを変換して座標を埋める。

## 5. 画面仕様（ボトムタブ5つ）

> 5タブが窮屈なら「イベント」をホームに統合して4タブにしてもよい。

### ① ホーム（index）
- ヘッダー：アプリ名「いいだっこ」＋ りんごモチーフのロゴ
- **緊急バナー**（赤系）：「こどもの急病 → ♯8000」タップで電話発信
- **大ボタン**：「あずける・あそぶ場所（マップ）」「今月のイベント」「相談する」
- 新着のお知らせ 3〜5件（カード）
- 手帳PDF：みんなで子育てナビ / パパナビ / まご手帳（公式PDFへリンク）

### ② さがす（search）― サイトを見やすく
- 上部に **セグメント切替 `[ 年齢でさがす | 目的でさがす ]`**
- **年齢**：妊娠〜出産 / 0〜1歳 / 2〜3歳 / 4〜6歳（2×2カード）
- **目的**：こどもを預ける / 支援・手当 / 入園 / おでかけ / 健診と学級 / 子育てに悩んだら（6カード）
- カードタップ → 記事リスト（タイトル・更新日・1〜2行の要点 ＋ 公式/PDFへの外部リンク）
- PDFで配られている内容は要点をカード化し、本文は公式PDFをリンクで開く

### ③ マップ（map）★アプリの主役
- 上部右に **切替トグル `[ 地図 | 一覧 ]`**
- **カテゴリ絞り込みチップ**（横スクロール）：すべて / 保育園 / 認定こども園 / 一時預かり / つどいの広場 / こども食堂 / 遊び場
- **地図モード**：施設をカテゴリ色ピンで表示。ピンタップで下部に施設カードがせり出す
- **一覧モード**：施設カードの縦リスト
- 施設カード：種別タグ・対象年齢タグ・名称・地区・開所時間 ＋ `[地図で見る][電話][公式]`

### ④ イベント（events）
- 日付順カードリスト。各カードに 日付タグ・対象年齢タグ・カテゴリタグ ＋ 場所 ＋ 申込/詳細リンク
- カテゴリ（こども食堂/講座/ひろば）でピル絞り込み。過ぎたものは非表示またはグレーアウト

### ⑤ きんきゅう（emergency）
- ♯8000（長野県小児救急電話相談・毎日19:00〜翌8:00）→ 発信ボタン
- 休日夜間当番医ページ・「こどもの急病 症状別対応」PDF へのリンク
- 医療情報は公式の言い回しを尊重し改変しない

## 6. データ構造（静的）

```ts
// types.ts
export type AgeBand = 'pregnancy' | 'age0_1' | 'age2_3' | 'age4_6';
export type Purpose = 'care' | 'support' | 'enroll' | 'outing' | 'checkup' | 'worry';
export type FacilityCat =
  | 'hoiku' | 'kodomoen' | 'ichiji' | 'hiroba' | 'shokudo' | 'asobi';

// さがす：サイトの情報カード
export interface Article {
  id: string;
  title: string;
  updatedAt: string;       // 'YYYY-MM-DD'
  ages: AgeBand[];
  purposes: Purpose[];
  summary?: string;        // 1〜2行（PDFの要点を平易に）
  url: string;             // 公式ページ or PDF or 申込フォーム
  source: '飯田市子育て応援サイト';
}

// マップ：施設
export interface Facility {
  id: string;
  name: string;
  category: FacilityCat;
  area: string;            // 地区
  address: string;
  lat: number;             // ビルド前にジオコーディングして直書き
  lng: number;
  ages?: string;           // '0〜5歳' 等
  hours?: string;          // '7:30〜18:30' 等
  tel?: string;
  url?: string;
  note?: string;           // '要事前申込' 等
  source: '飯田市子育て応援サイト';
}

// イベント
export interface EventItem {
  id: string;
  title: string;
  date: string;            // 表示 '6/14(日)'
  sortKey: string;         // 'YYYY-MM-DD'
  place: string;
  lat?: number; lng?: number;
  ages: AgeBand[];
  category: 'food' | 'lesson' | 'hiroba' | 'other';
  needsApply: boolean;
  url: string;
}
```

### デモ用サンプル（実サイトから／各カテゴリ数件で十分）
- **さがす（記事）**：授乳・育児相談、児童手当、産後ケア、休日保育・延長保育の案内、乳幼児健診
- **施設**：飯田子供の園保育園（橋北）・慈光保育園（東野）・座光寺保育園・丸山保育園・千代保育園／わいわいひろば（飯田短大）・ゆいきっず広場／おあがりて食堂（上郷公民館）
- **イベント**：わくわく並木広場(6/14)・ゆいきっず講座 絵本のよみきかせ(6/4)・親子フラ体験会・BP1"赤ちゃんがきた!"
- **緊急**：♯8000・休日夜間当番医

> すべて出典は飯田市公式サイト。各カードは公式URLへ遷移できること。

## 7. フェーズ計画

- **Phase 1（今回）**：静的データ＋地図の見た目重視デモ。商工会議所・役所への提示用。
- **Phase 2（採用後）**：データ更新の仕組みを設計。(a) Supabase で自分がCMS入力 / (b) 公式RSS（`/rss/10/site-947.xml`）で半自動取得。運用主体を役所と相談。
- **Phase 3（事業化）**：他自治体への横展開（同型テンプレ）。「お医者さんナビ」と統合した地域プラットフォーム化も視野。

## 8. 運用・権利まわりの前提（役所相談で必ず確認）

- 本アプリは公式情報の**二次案内**。情報は改変せず、出典明記＋公式ページへ誘導する。
- PDF・画像・ロゴの転載はしない（リンクで開く）。
- 既存公式アプリ「**母子モ**」とは役割をすみ分け（母子モ＝記録／いいだっこ＝見やすく・場所を探す）。
- 公開はベータ（限定配布）から。正式公開前に役所の許可・情報提供の合意を取る。
- Google Maps API の利用規約・キー管理に留意。実行時ジオコーディングをしない設計でコストを抑える。
